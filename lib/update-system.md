# 🏗️ UGC系统更新体系

## 📋 体系设计原则

### **1. 以网站为中心**
- 所有UGC组件必须使用网站的**设计系统**
- 颜色、字体、间距、动画必须与网站**完全一致**
- 语言系统必须与网站**完全集成**

### **2. 无缝衔接**
- 布局必须与前后section**自然过渡**
- 不能有空白断层或割裂感
- 交互体验必须与网站**完全一致**

### **3. 可持续更新**
- 建立**配置系统**，便于后续更新
- 组件设计要**模块化**，便于维护
- 建立**版本控制**和**回滚机制**

## 🎨 设计系统集成

### **颜色系统**
```typescript
// 必须使用网站的颜色系统
stone: { 50, 100, 200, 300, 400, 500, 600, 700, 800, 900 }
amber: { 50, 100, 200, 300, 400, 500, 600, 700, 800, 900 }
```

### **间距系统**
```typescript
// 必须使用网站的间距系统
section: 'py-32'    // 与网站section间距一致
container: 'px-6'   // 与网站容器间距一致
card: 'p-6'         // 与网站卡片间距一致
title: 'mb-6'       // 与网站标题间距一致
```

### **字体系统**
```typescript
// 必须使用网站的字体系统
title: 'text-4xl md:text-5xl font-serif'
subtitle: 'text-lg text-stone-600'
body: 'text-stone-700 leading-relaxed'
```

## 🚀 快速更新指南

### **添加新UGC功能**
1. 在 `lib/ugc-config.ts` 中添加配置
2. 在 `components/UGCIntegrated.tsx` 中实现
3. 使用网站的设计系统类名
4. 集成网站的语言系统

### **修改现有功能**
1. 只修改配置，不修改组件结构
2. 确保与网站设计系统一致
3. 测试布局衔接

### **紧急修复**
1. 使用最小改动原则
2. 优先保证网站可访问性
3. 后续进行系统性修复

## 🔍 问题诊断

### **如果出现割裂感**
1. 检查section的class是否与网站一致
2. 验证背景颜色和渐变
3. 检查间距是否匹配

### **如果出现空白断层**
1. 检查前后section的padding/margin
2. 验证容器宽度是否一致
3. 检查是否有额外的空白div

### **如果语言不一致**
1. 检查是否使用网站的语言系统
2. 验证localStorage语言设置
3. 检查翻译文本是否完整
